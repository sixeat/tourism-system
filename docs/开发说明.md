# 智旅旅行系统开发说明

## 1. 项目定位

本项目是一个基于 Java SSM 的旅游推荐与预订系统，面向普通游客和后台管理员两类角色。系统围绕“路线规划、酒店预订、景点门票、订单管理、AI 问答、地图展示”形成完整旅游服务闭环。

项目名称：智旅旅行  
技术架构：Spring + Spring MVC + MyBatis + JSP + MySQL + Tomcat  
部署方式：Maven 打包 WAR 后部署到 Tomcat 9

## 2. 技术栈

1. 后端框架：Spring、Spring MVC、MyBatis
2. 前端页面：JSP、HTML、CSS、JavaScript
3. 数据库：MySQL
4. 地图服务：高德地图 Web JavaScript API
5. AI 问答：本地 RAG 检索问答，预留 OpenAI 兼容接口
6. 运行容器：Tomcat 9
7. 构建工具：Maven

## 3. 系统角色

### 3.1 普通用户

普通用户可以完成注册、登录、浏览路线、查看景点、预订酒店和门票、支付订单、取消订单、评价订单、收藏路线/酒店/景点、使用 AI 旅游问答。

### 3.2 管理员

管理员可以进入后台管理页，维护景点、酒店、门票、路线等基础资源，并查看订单和统计数据。

## 4. 功能模块

### 4.1 用户认证模块

1. 用户登录
2. 新用户注册
3. 当前登录用户查询
4. 退出登录
5. Session 登录状态维护
6. 普通用户与管理员角色区分

相关接口：

1. `POST /api/auth/login`
2. `POST /api/auth/register`
3. `GET /api/auth/me`
4. `POST /api/auth/logout`

### 4.2 首页与路线推荐模块

用户首页提供目的地、出发日期、游玩天数、人均预算、兴趣关键词等输入项。系统根据用户输入推荐旅游路线，并展示路线卡片、每日安排、费用明细和地图线路。

主要功能：

1. 按城市、预算、天数、兴趣推荐路线
2. 展示路线名称、主题、预算、景点顺序
3. 展示路线详情，包括每日行程和费用拆分
4. 右侧展示目的地灵感和预算分析
5. 路线可加入收藏

相关接口：

1. `POST /api/route/recommend`

### 4.3 景点与门票模块

系统展示景点列表，支持查看景点对应门票，并可选择门票加入快速下单区域。

主要功能：

1. 景点列表展示
2. 景点评分、分类、描述展示
3. 门票列表查询
4. 门票库存展示
5. 门票选择与下单联动
6. 景点收藏

相关接口：

1. `GET /api/scenic/list`
2. `GET /api/ticket/list?scenicId=1`

### 4.4 酒店预订模块

系统按城市展示酒店，支持查看酒店房型，并选择房型创建酒店订单。

主要功能：

1. 酒店列表查询
2. 房型列表查询
3. 房型库存展示
4. 酒店订单创建
5. 酒店收藏

相关接口：

1. `GET /api/hotel/list?city=杭州`
2. `GET /api/hotel/rooms?hotelId=1`
3. `POST /api/order/hotel/create`

### 4.5 订单模块

订单模块统一展示酒店订单和门票订单，支持状态筛选、订单取消、模拟支付和订单评价。

主要功能：

1. 我的订单列表
2. 酒店订单和门票订单统一展示
3. 按订单类型和订单状态筛选
4. 未支付订单取消
5. 模拟支付，状态从 `CREATED` 变为 `PAID`
6. 已支付订单可填写星级和评价

相关接口：

1. `GET /api/user/orders`
2. `POST /api/order/hotel/cancel?orderId=1`
3. `POST /api/order/ticket/cancel?orderId=1`
4. `POST /api/order/hotel/pay?orderId=1`
5. `POST /api/order/ticket/pay?orderId=1`

### 4.6 AI 旅游问答模块

AI 问答模块使用本地 RAG 检索方式，从系统中的景点、酒店、门票、路线和订单规则中检索相关资料，并生成中文回答。

主要功能：

1. 支持旅游路线咨询
2. 支持门票价格咨询
3. 支持酒店推荐咨询
4. 支持订单规则咨询
5. 展示回答来源和追问建议
6. 预留外部大模型接口，失败时自动回退本地 RAG

相关接口：

1. `POST /api/ai/ask`

### 4.7 高德地图模块

首页接入高德地图，展示景点位置，并根据推荐路线绘制景点顺序。

主要功能：

1. 景点地图标记
2. 景点名称标签
3. 路线折线展示
4. 地图加载失败兜底提示
5. 路线卡片与地图联动

配置文件：

`src/main/webapp/static/js/amap-config.js`

### 4.8 收藏与个人中心模块

首页个人中心展示当前登录用户信息，并提供订单入口、AI 助手入口和收藏列表。

主要功能：

1. 显示当前用户名和角色
2. 路线、酒店、景点收藏
3. 收藏列表展示
4. 收藏移除

说明：收藏数据已保存到数据库 `user_favorite` 表，可跨浏览器和重新登录保留。

### 4.9 后台管理模块

后台管理页面向管理员，支持基础资源维护和订单管理。

主要功能：

1. 后台统计概览
2. 景点增删改查
3. 酒店增删改查
4. 门票增删改查
5. 路线增删改查
6. 订单列表查看
7. 订单状态维护

相关接口：

1. `GET /api/admin/dashboard/summary`
2. `GET /api/admin/scenic/list`
3. `POST /api/admin/scenic/save`
4. `PUT /api/admin/scenic/update`
5. `DELETE /api/admin/scenic/delete/{id}`
6. `GET /api/admin/hotel/list`
7. `POST /api/admin/hotel/save`
8. `PUT /api/admin/hotel/update`
9. `DELETE /api/admin/hotel/delete/{id}`
10. `GET /api/admin/ticket/list`
11. `POST /api/admin/ticket/save`
12. `PUT /api/admin/ticket/update`
13. `DELETE /api/admin/ticket/delete/{id}`
14. `GET /api/admin/route/list`
15. `POST /api/admin/route/save`
16. `PUT /api/admin/route/update`
17. `DELETE /api/admin/route/delete/{id}`
18. `GET /api/admin/order/list`

## 5. 页面入口

1. 入口页：`/index.jsp`
2. 登录/注册页：`/login.jsp`
3. 用户首页：`/home.jsp`
4. 我的订单页：`/orders.jsp`
5. 后台管理页：`/admin.jsp`

## 6. 默认账号

1. 普通用户：`demo / demo123`
2. 管理员：`admin / admin123`

## 7. 数据库说明

核心数据表包括：

1. `sys_user`：用户表
2. `scenic_spot`：景点表
3. `hotel`：酒店表
4. `hotel_room`：酒店房型表
5. `ticket`：门票表
6. `travel_route`：旅游路线表
7. `route_spot`：路线景点关联表
8. `hotel_order`：酒店订单表
9. `ticket_order`：门票订单表

初始化脚本：

`src/main/resources/sql/init.sql`

## 8. 运行步骤

1. 创建 MySQL 数据库：`tourism_system`
2. 执行初始化 SQL：`src/main/resources/sql/init.sql`
3. 修改数据库配置：`src/main/resources/application.properties`
4. 执行 Maven 打包：`mvn clean package`
5. 将 `target/tourism-system.war` 部署到 Tomcat 9
6. 浏览器访问：`http://localhost:8080/tourism-system/`

## 9. 项目亮点

1. 前台、后台、订单、地图、AI 问答功能较完整
2. 旅游路线支持预算、天数、兴趣条件推荐
3. 路线详情包含每日安排和费用拆分，适合答辩展示
4. 高德地图可视化展示景点和路线顺序
5. AI 问答采用 RAG 思路，能结合系统数据回答问题
6. 订单支持创建、取消、模拟支付、评价，形成业务闭环
7. 页面统一中文 UI，适合课堂演示

## 10. 可继续优化方向

1. 继续扩展评价审核、收藏统计和用户画像分析
2. 增加更多城市、景点、酒店和路线数据
3. 增加退款、发票、优惠券等订单扩展功能
4. 接入可用的大模型 API，提高 AI 问答生成质量
5. 增加用户资料编辑和头像上传功能
